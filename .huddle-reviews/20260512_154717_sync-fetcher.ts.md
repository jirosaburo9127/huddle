## Codex レビュー

- [優先度: 今すぐ] `reactions` の差分判定が ID 集合だけなので、同じ reaction ID の中身（emoji/type/user/status など）が変わっても更新されません。`reactions` は `JSON.stringify` 相当の安定比較、または `id + updated_at + type/emoji/user_id` など必要フィールドまで比較してください。

- [優先度: 今すぐ] `f in fresh_` は継承プロパティも拾います。DB row 由来なら通常問題化しにくいですが、汎用関数なので `Object.prototype.hasOwnProperty.call(fresh_, f)` に置き換える方が安全で意図も明確です。

- [優先度: 後で] `updateFields` が `map` の各要素ごとに再生成されています。関数外または `map` の前に `const updateFields = [...] as const` として定義すると、可読性・型安全性・無駄な allocation の面で改善できます。

- [優先度: 後で] テストを追加してください。最低限、既存 ID の `content/deleted_at/status/reply_count/reactions` が更新されること、変更なしなら同じ `prev` 参照を返すこと、reaction の中身変更が反映されること、新規追加時だけ `created_at` 昇順になることを押さえるとよいです。
