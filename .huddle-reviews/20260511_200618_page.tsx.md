## Codex レビュー

- [優先度: 今すぐ] `useEffect` 内で `setLoading(false)` が `wsMembers` 取得以降の失敗時に保証されていません。`try/finally` にして、`getUser` / workspace / members / DM取得のどこで失敗しても最後に `setLoading(false)` される形にしてください。

- [優先度: 今すぐ] `profiles` が配列かつ空配列の場合、`profiles: m.profiles[0]` が `undefined` になり、`WsMember` 型と実データがずれます。`const profile = Array.isArray(...) ? ...[0] : ...` として、`if (!profile) return null` で落としてから `filter(Boolean)` してください。

- [優先度: 後で] `workspace_members` の取得対象に自分自身が含まれる可能性があります。`CreateDmModal` 側で除外していないなら、ここで `.neq("user_id", user.id)` を追加するか、`setMembers` 前に `user_id !== user.id` で除外してください。

- [優先度: 後で] `workspace_members` 取得エラーを無視しているため、権限/RLS/ネットワーク失敗時に空状態と区別できません。`const { data, error } = ...` にして、少なくとも `console.error` または既存のエラー表示パターンに流してください。


## Claude Code 修正 (応答)

**採用しなかった指摘:**
- 指摘3 (自分自身を除外): `CreateDmModal` 内ですでに `m.user_id !== currentUserId` で除外済み (line 46)。二重防御より単一責任を優先。

**変更サマリー:**
- `useEffect` 全体を `try/finally` でくくり、どこで早期 return / 例外が起きても `setLoading(false)` が必ず走る形に統一。
- `profiles` 正規化時に空配列 → `undefined` を弾く形に書き換え、型ガード `(m): m is WsMember` で `null` を除外。
- `workspace_members` 取得時に `error` を分割代入し、失敗時は `console.error` を出力。
