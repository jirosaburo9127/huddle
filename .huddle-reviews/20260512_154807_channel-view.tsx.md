## Codex レビュー

- [優先度: 今すぐ] `user.id` と `currentUserId` の不一致を検証してください。`getUser()` 後に `user.id !== currentUserId` なら楽観表示を取り消して return する、または insert/optimistic ともに `user.id` を使うべきです。現在は別ユーザーIDで楽観表示・DB insert し得ます。

- [優先度: 今すぐ] `supabase.auth.getUser()` の `error` を見ていません。`const { data, error: authError } = ...` とし、`authError || !data.user` の場合は rollback してください。ネットワーク/セッション異常時の扱いが不明確です。

- [優先度: 後で] 未認証 rollback 時に `setReplyTo(null)` 済みの返信状態が復元されません。`parentIdSnapshot` がある場合、認証失敗時は元の `replyTo` を保持するか、認証確認を `setReplyTo(null)` より前に移してください。

- [優先度: 後で] `profiles.email` と `display_name` の fallback が消えており、`myProfile` 未ロード時に送信者表示が空になります。`user.email` / `user.user_metadata?.display_name` の fallback は `getUser()` 後に楽観メッセージを補正するか、既存 fallback を維持してください。

- [優先度: 後で] テスト追加対象です。`getUser` 成功、未認証、`currentUserId` 不一致、`myProfile` 未ロード時の楽観表示、rollback 後にメッセージが残らないことを最低限カバーしてください。
