## Codex レビュー

- [優先度: 今すぐ] `stabilizeJumpPosition` の戻り値 `cleanup` が呼び出されない経路があると、`ResizeObserver` / timer / event listener が最大4.2秒残ります。呼び出し側で `useEffect` cleanup や次回ジャンプ前に必ず前回 cleanup を実行する形にしてください。

- [優先度: 今すぐ] `handleJumpToMessage` の highlight 削除 `setTimeout` が unmount 後も残ります。既存問題の拡大ですが、ジャンプ処理が増えるなら timeout id を管理して unmount / 次回ジャンプ時に clear してください。

- [優先度: 後で] `container.addEventListener("keydown", cleanup)` は scroll container にフォーカスが無いと効かず、PageUp/PageDown/矢印キー操作では解除されない可能性があります。`document` 側で監視するか、実際のスクロールイベントを見て解除するほうが堅いです。

- [優先度: 後で] `ResizeObserver` が未定義の環境で落ちます。テスト環境や古いブラウザ対策として `typeof ResizeObserver === "undefined"` の場合は timer ベースのみで動く fallback を入れてください。

- [優先度: 任意] テスト追加: 「画像ロード等で高さが変わった後も対象メッセージが中央付近に残る」「wheel/touch/keyboard 操作で再固定が止まる」「unmount 時に observer/timer が cleanup される」をカバーすると回帰を防げます。
