## Codex レビュー

- [優先度: 今すぐ] `supabase.auth.getUser()` の失敗時に未処理です。`const { data: { user }, error } = await supabase.auth.getUser();` として、`error` または `!user` の場合は明示的に `setUnreadBreakdown(...)` を初期値へ戻すなど、前回表示の未読数が残らないようにしてください。

- [優先度: 後で] `mark_*_seen` 後の `if (cancelled) return;` は state 更新防止としては妥当ですが、RPC の失敗を無視して既読扱いの UI 更新に進む可能性があります。`const { error } = await supabase.rpc(...)` を確認し、失敗時は `loaded` やドット消去を進めない、またはログ出力する形にしてください。

- [優先度: 任意] `getUser()` の取得処理が unread breakdown 側だけに追加されています。下の activity 読み込み側でもユーザー前提の RPC があるなら、同じ形の早期 return / エラー処理を共通化して可読性を上げると、認証切れ時の挙動が揃います。
