## Codex レビュー

- [優先度: 今すぐ] `useEffect` の cleanup で `jumpActiveRef.current` を戻していないため、`?m=` が外れる/別URLへ遷移する/`channel.id` が変わるタイミングで cleanup だけ走ると、自動スクロール抑止が `true` のまま残る可能性があります。`return () => { ... }` 内で、少なくとも `jumpHandledIdRef.current === target` の場合は `jumpActiveRef.current = false` も実行してください。

- [優先度: 後で] `jumpHandledIdRef.current = target` を effect 冒頭で立てる設計だと、DOM待ちに失敗した後に同じ `?m=<id>` のリンクを再クリックしても、URLが変わらなければ effect が再実行されず「次回タップで再試行」にならない可能性があります。失敗時にクエリを一度消す/クリック側で明示的に再試行関数を呼ぶ/`m` に nonce を付ける、など同一URLでも再試行できる形にしてください。

## 採用判断

**[今すぐ]** ✅ 採用 — cleanup で `jumpActiveRef.current` を戻さないと自動スクロールが永久抑止されるリスクは実在するため修正。`jumpHandledIdRef.current === target` のときだけ戻すガード付きで対応。

**[後で]** ⏸ 見送り — 妥当な指摘だが本人が「後で」優先度。同一URL再試行は nonce / クエリリセット / クリックハンドラ側 retry のどの設計を取るか方針判断が必要なので、Author 確認待ちとして今回は触らず。

## 変更サマリ

- `?m=<id>` ジャンプ effect の cleanup に「担当中のジャンプなら `jumpActiveRef.current = false` に戻す」処理を追加。
- これにより releaseTimer 未発火のまま URL 変更 / channel 切替 / unmount しても自動スクロール抑止フラグが残らない。
