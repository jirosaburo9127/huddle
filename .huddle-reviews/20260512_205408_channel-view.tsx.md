## Codex レビュー

- [優先度: 今すぐ] `handleReturnToJumpTarget` が `executeJump()` の戻り値を無条件に `jumpReturnCleanupRef.current` に代入しています。`executeJump` がジャンプ失敗時に `undefined` / `false` を返す可能性があるなら型・実行時ともに壊れます。`const cleanup = executeJump(...); if (cleanup) jumpReturnCleanupRef.current = cleanup;` のように戻り値をガードしてください。

- [優先度: 今すぐ] `executeJump(jumpReturnTargetId)` に置き換えたことで、戻るボタン押下時にも `jumpReturnTargetId` や表示タイマーを再設定する副作用が走る可能性があります。戻る操作では「戻るボタン自体を消す / 再表示しない」挙動にしたいはずなので、`executeJump(targetId, { suppressReturnTarget: true })` のようなオプションを追加するか、戻る専用処理で必要なジャンプだけ実行してください。

- [優先度: 後で] `jumpReturnCleanupRef` と `stabilizeCleanupRef` の責務が近く、どちらがどの cleanup を保持するのか読み取りづらくなっています。`executeJump` が返す cleanup の中身に合わせて、`jumpExecutionCleanupRef` など実体が分かる名前にするか、同一種類の cleanup なら ref を統合してください。
