## Codex レビュー

- [優先度: 今すぐ] `setInterval` のクリーンアップがありません。`?m=` 変更・アンマウント・再実行時に古いリトライが残り、別IDのジャンプ中に `jumpHandledIdRef` / `jumpActiveRef` を古い値で更新し得ます。`useEffect` 内で `let retryInterval: ReturnType<typeof setInterval> | null` を保持し、cleanup で `clearInterval`、さらに `target` 一致確認を入れてください。

- [優先度: 今すぐ] `jumpHandledIdRef.current` を成功時までセットしないため、同じ `target` の effect が再発火すると並列リトライが走る可能性があります。`jumpPendingIdRef` を追加するか、開始時に `jumpHandledIdRef.current = target` 相当の「処理中」状態を立て、失敗時だけ戻す実装にしてください。

- [優先度: 後で] 成功後の `jumpActiveRef` 解除が `500ms` だと、既存コメントにある ResizeObserver の最大 `3s` より早く、自動スクロールに負ける可能性があります。旧実装と同じく `3100ms` 付近まで再ジャンプするか、少なくとも抑止解除を `3500ms` 程度に戻してください。


## Claude Code 修正 (応答)

3指摘とも採用。

**変更点（3行以内）:**
- useEffect の cleanup を追加し `clearInterval`/`clearTimeout` で古いリトライを停止。`cancelled` フラグと `jumpHandledIdRef.current !== target` チェックで別IDジャンプとの競合を遮断。
- 開始時に `jumpHandledIdRef.current = target` を立てて並列リトライを防止。失敗パス（対象なし／タイムアウト）でのみ `null` に戻して再試行可能に。
