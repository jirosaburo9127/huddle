## Codex レビュー

- [優先度: 後で] 成功時の `router.replace` が effect 開始時点の `searchParams` を使っているため、ジャンプ処理中に他のクエリが更新されると古い値で上書きする可能性があります。成功直前の URL から作る形にすると安全です: `const url = new URL(window.location.href); url.searchParams.delete("m"); router.replace(url.search ? \`${url.pathname}${url.search}\` : url.pathname, { scroll: false })`。

- [優先度: 後で] 今回の意図である「`?m=` だけ消して他クエリを保持」「`?m=` 消滅後に同じ投稿へ再ジャンプ可能」をテストで固定したいです。例: `?m=abc&foo=bar` からジャンプ成功後に `?foo=bar` になること、同じ `m=abc` を再付与したとき `jumpHandledIdRef` に阻まれないこと。
