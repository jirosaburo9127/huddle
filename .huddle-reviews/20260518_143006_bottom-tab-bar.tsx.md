## Codex レビュー

- [優先度: 今すぐ] 非表示の「その他」メニューが常時マウントされており、`aria-hidden` / `pointer-events-none` だけなのでキーボードフォーカスで中の `Link` / `button` に到達できます。`showMore` が false の時は `inert` を付ける、または各操作要素に `tabIndex={-1}` を付けてフォーカス不可にしてください。

- [優先度: 今すぐ] `messageInputFocused` 時の `<nav aria-hidden>` 配下にもフォーカス可能な `button` / `Link` が残ります。アクセシビリティ的に不正で、キーボード操作では隠れたタブバーに到達し得ます。`inert={messageInputFocused}`、または子要素の `tabIndex` / `disabled` 制御を追加してください。

- [優先度: 後で] アクティブ判定が `pathname.includes(...)` ベースなので誤判定しやすいです。特に `/${workspaceSlug}/search` は `isHome` から除外されておらず、検索画面で「ホーム」が選択状態になります。`pathname === basePath` / `pathname.startsWith(`${basePath}/calendar`)` のように workspace 配下の明示的なルート判定に寄せ、`search` も More 側として扱ってください。

- [優先度: 後で] 外側タップ判定で `document.querySelector("[data-more-popover]")` / `trigger` を毎回グローバル検索しています。同一画面に複数インスタンスが出た場合や将来の重複で誤動作します。`useRef<HTMLDivElement | HTMLButtonElement>` を使い、対象要素をコンポーネント内に閉じてください。

- [優先度: 任意] 回帰防止として、`showMore=false` 時にメニュー項目へ Tab 移動できないこと、`/search` / `/dm-list` / `/files` / `/albums` でホームが active にならないことを Testing Library で追加するとよいです。
